# Chronicles of Dusk（黄昏编年史）—— Java 2D 生存动作游戏 架构详解

---

## 一、项目总览

《Chronicles of Dusk》是一款 **Java Swing 纯手写** 的 2D 像素风生存动作游戏（类似《吸血鬼幸存者》Vampire Survivors）。整个项目约 **4000+ 行代码**，**31 个 Java 源文件**，无需任何第三方依赖（无 Maven/Gradle），仅使用 JDK 标准库（`java.awt`、`javax.swing`、`javax.sound.sampled`）。

### 核心玩法
- 从 3 个职业中选择一个，在开放地图中生存
- 玩家自动攻击最近的敌人，敌人从屏幕边缘不断生成
- 击杀敌人获得经验值（XP），升级后三选一强化（新武器/武器升级/属性加成）
- 每 25 级进入新的 **Tier（位阶）**，属性大幅提升，Tier 3 解锁 **终极技能**
- 坚持 5 分钟后 **Boss 登场**，击败 Boss 获胜

### 技术特点
| 特性 | 实现方式 |
|------|----------|
| 渲染 | `java.awt.Graphics2D` + 双缓冲 `JPanel` |
| 游戏循环 | `javax.swing.Timer`，默认 60 FPS |
| 音频 | `javax.sound.sampled.Clip`，支持循环 BGM + 音效 |
| 碰撞检测 | 圆形-圆形 / 圆形-AABB，空间哈希网格优化 |
| 精灵动画 | 精灵表（sprite sheet）帧动画，多方向+多动作 |
| 状态管理 | 11 种游戏状态（菜单、选角、战斗中、升级暂停等） |

## 三、游戏状态机

游戏一共有 **11 种状态**，定义在 `SurvivalGame` 中：

```
STATE_MENU (0)        → 主菜单
STATE_CHAR_SELECT (1) → 角色选择
STATE_INTRO (9)       → 开场剧情动画
STATE_PLAYING (2)     → 正常战斗中
STATE_UPGRADE_PAUSE(3) → 升级暂停（三选一）
STATE_TIER_UP (8)     → 位阶提升过场
STATE_BOSS_INTRO (4)  → Boss 登场警告
STATE_WEAPON_SWAP (7) → 武器替换选择
STATE_VICTORY (5)     → 胜利结算
STATE_DEFEAT (6)      → 失败结算
STATE_PAUSED (10)     → 暂停
STATE_HELP (11)       → 帮助/按键说明
```

状态流转：
```
MENU → INTRO → CHAR_SELECT → PLAYING ⇄ UPGRADE_PAUSE
                                  ↓         ↑
                              TIER_UP ────┘
                                  ↓
                              BOSS_INTRO → PLAYING → VICTORY/DEFEAT → CHAR_SELECT
                              PLAYING → PAUSED → PLAYING
                              PLAYING → WEAPON_SWAP → PLAYING
```

## 四、逐文件详细分析

### 4.1 核心引擎层

#### **GameEngine.java** (981 行) — 游戏引擎抽象基类

**角色：** 这是整个游戏的底层框架，提供了所有与 Java 图形/音频/输入系统的接口封装。

**关键内部结构：**

```
GameEngine
├── GamePanel (内部类, extends JPanel)  — 渲染画布
├── GameTimer (内部类, extends Timer)   — 游戏循环时钟
└── AudioClip (内部类)                  — 音频数据容器
```

**核心方法分类：**

1. **窗口管理** — `setupWindow()`, `setWindowSize()`, `width()`, `height()`
   - 创建 `JFrame`，注册键盘事件（通过 `KeyEventDispatcher`，可全局捕获按键）
   - 注册鼠标事件（点击、移动、拖拽）

2. **游戏循环** — `gameLoop(framerate)`, `measureTime()`
   - 使用 `javax.swing.Timer` 实现固定帧率循环
   - `measureTime()` 计算两帧之间的实际时间差（`dt`），用于帧率无关的物理更新
   - 每帧调用 `update(dt)` → `mPanel.repaint()` → `paintComponent()`

3. **渲染函数** — 线条、矩形、圆形、文字、图片
   - `drawImage()`, `drawLine()`, `drawRectangle()`, `drawCircle()` 等
   - `drawText()` 支持自定义字体和大小
   - 颜色管理（预设黑/白/红/绿/蓝等常用颜色）

4. **变换栈** — `saveCurrentTransform()`, `restoreLastTransform()`, `translate()`, `rotate()`, `scale()`
   - 使用 `Stack<AffineTransform>` 维护变换矩阵
   - 用于相机偏移、投射物旋转等

5. **音频系统** — `loadAudio()`, `playAudio()`, `startAudioLoop()`, `stopAudioLoop()`
   - 使用 `javax.sound.sampled.AudioSystem` 加载 WAV 文件
   - `AudioClip` 内部类：存储 `AudioFormat` + `byte[]` 音频数据
   - 支持循环播放（BGM）和音量控制（`FloatControl.Type.MASTER_GAIN`）

6. **数学工具** — `rand()`, `sqrt()`, `distance()`, `sin()`, `cos()`, `atan2()` 等
   - 封装 `Math` 类，角度使用**度数**（内部自动转换为弧度）

7. **图片操作** — `loadImage()`, `subImage()`
   - 使用 `ImageIO.read()` 加载 PNG
   - `subImage()` 从精灵表中切割单帧

**设计要点：**
- `update(double dt)` 和 `paintComponent()` 是抽象方法，由于类 `SurvivalGame` 实现
- 键盘事件通过 `KeyEventDispatcher` 全局捕获，不受焦点影响
- 构造方法使用 `SwingUtilities.invokeLater()` 确保 GUI 创建在 EDT 线程

### 4.2 游戏主控层

#### **SurvivalGame.java** (1781 行) — 游戏主逻辑类

**角色：** 整个游戏的"大脑"，管理所有子系统、游戏状态、实体列表，处理输入输出。

**关键字段：**
```java
int gameState;                    // 当前游戏状态
boolean[] keys = new boolean[512]; // 按键状态数组（按下=true, 松开=false）
Player player;                    // 玩家实例
ArrayList<Enemy> enemies;         // 敌人列表
ArrayList<Projectile> projectiles; // 投射物列表
ArrayList<Pickup> pickups;        // 掉落物列表
VfxManager vfx;                   // 特效管理器
Camera camera;                    // 相机
GameMap gameMap;                  // 地图
EnemySpawner spawner;             // 敌人生成器
CollisionSystem collisionSystem;  // 碰撞系统
WeaponManager weaponManager;      // 武器管理器
UpgradeSystem upgradeSystem;      // 升级系统
AudioManager audio;               // 音频管理器
```

**核心方法分析：**

1. **`init()`** — 游戏初始化
   - 创建所有子系统实例
   - 初始化精灵管理器 `SpriteManager.init(this)`

2. **`update(double dt)`** — 每帧调用
   - 处理音频状态切换（根据 `gameState` 变化自动切换 BGM）
   - 根据当前状态分发更新：
     - `STATE_PLAYING` → `updatePlaying(dt)`
     - `STATE_UPGRADE_PAUSE` → 更新悬停检测
     - `STATE_BOSS_INTRO` → 倒计时
     - `STATE_TIER_UP` → 倒计时 + 终极技能更新
     - `STATE_INTRO` → 剧情动画更新
     - 其他状态 → 仅更新特效

3. **`updatePlaying(dt)`** — 核心战斗逻辑
   ```
   检查玩家存活 → 玩家更新 → 相机跟随 → 敌人生成 → 武器开火
   → 所有实体更新 → 碰撞检测 → 清理死亡实体 → 特效更新
   ```

4. **`renderWorld()`** — 世界渲染
   - 应用相机变换（`vfx.beginCamera()` → 平移世界坐标到屏幕坐标）
   - 渲染顺序：地面 → 地图障碍物 → 掉落物 → 敌人 → 投射物 → 玩家 → 特效
   - 恢复相机变换（`vfx.endCamera()`）
   - 渲染 HUD

5. **`renderHUD()`** — 抬头显示
   - 左上角：HP 条、XP 条、等级、时间
   - 终极技能冷却条（Tier 3+）
   - 里程碑进度（25/50/75 级标记）
   - 武器栏位（最多 6 个，按稀有度着色）
   - Boss 生命条（Boss 存在时）
   - 右上角：分数面板

6. **`renderMenu()`** — 主菜单渲染
   - 深色背景 + 竖线纹理
   - "CHRONICLES OF DUSK" 标题（带浮动动画）
   - 四个按钮：START / SETTINGS / CONTROLS / EXIT
   - 设置面板（屏幕震动开关、战斗节奏选择）

7. **`renderCharSelect()`** — 角色选择界面
   - 三张角色卡片：Warrior / Mage / Assassin
   - 每张卡片显示：头像圆章、职业名、HP/SPD/DMG 倍率、特色描述
   - 卡片悬停上浮动画

8. **`renderUpgradeOverlay()`** — 升级三选一界面
   - 半透明遮罩 + 三张升级卡片
   - 显示升级名称和描述

9. **`renderEndScreen(boolean victory)`** — 结算界面
   - 分数计算：击杀分 + 生存时间分 + Boss 击杀奖励 + 满血通关奖励
   - 等级里程碑倍率（Lv25: ×1.5, Lv50: ×2.0, Lv75: ×3.0）

10. **`renderIntro()`** — 开场剧情动画
    - 分 4 个阶段，总计约 22 秒
    - Phase 0：标题出现（1.5 秒）
    - Phase 1：滚动叙事文字（5.5 秒）
    - Phase 2：英雄召唤（7.5 秒）
    - Phase 3：最终号召 + 白屏过渡（7.5 秒）
    - 视觉元素：渐变天空、星空闪烁、月亮、山脉剪影、城堡像素画、火炬火焰动画、石中剑

11. **`startRun(CharacterDef def)`** — 开始一局游戏
    - 重置所有状态、清空实体列表
    - 创建玩家、生成地图、重置生成器和升级系统
    - 分配初始武器

12. **输入处理** — `keyPressed()`、`keyReleased()`、`mouseMoved()`、`mouseClicked()`
    - WASD 移动（通过 `keys[]` 数组持续检测）
    - SPACE 释放终极技能
    - P 暂停/恢复
    - ESC 返回/退出
    - 1/2/3 选择升级或武器替换
    - L 调试：快速升级

13. **辅助方法**
    - `findNearestEnemy(x, y)` — 找最近敌人（用于锁定目标武器）
    - `addEnemy()`, `addProjectile()`, `addPickup()` — 实体添加（带数量上限检查）
    - `enterWeaponSwap()` / `resolveWeaponSwap()` — 武器替换流程
    - `enterTierUp()` — 位阶提升（创建庆祝粒子）
    - `spawnEnemyNear()` — 在指定位置附近生成敌人
    - `spawnBossRadial()` — Boss 半血后释放环形弹幕
    - `calculateFinalScore()` — 最终分数计算

### 4.3 实体层

#### **Entity.java** (72 行) — 抽象实体基类

所有游戏对象的基类。关键设计：

```java
double x, y;        // 位置
double vx, vy;      // 速度
double radius;      // 碰撞半径（圆形碰撞体）
double width, height; // 矩形尺寸（障碍物用）
double maxHealth, health; // 生命值
boolean alive;       // 存活标记
int hitFlashTimer;   // 受击闪烁计时器
Color baseColor;     // 基础颜色
```

**模板方法模式：**
- `update(dt)` 是 `final` 的，自动执行位置更新（`x += vx * dt`）+ 闪烁计时
- `onUpdate(dt)` 是抽象的，子类实现各自的逻辑
- `takeDamage()` 包含通用受伤逻辑（扣血 → 闪烁 → 死亡检测 → `onDeath()` 回调）

**碰撞距离计算：** `distSqTo(other)` 返回距离平方（避免开方运算）

#### **Player.java** (183 行) — 玩家

**关键属性：**
```java
double facingAngleRad;    // 面朝方向（弧度）
double pickupRangeMultiplier; // 拾取范围倍率
double damageMultiplier;  // 伤害倍率
double cooldownMultiplier; // 冷却倍率
int tier;                 // 位阶（1-4）
UltimateSkill ultimate;   // 终极技能
```

**`onUpdate(dt)` 逻辑：**
1. 读取 WASD 按键 → 计算方向向量 → 归一化 → 乘以移动速度
2. 更新 `facingAngleRad`（面朝方向）
3. 无敌计时器衰减（受伤后短暂无敌 0.3 秒）
4. 动画计时器更新
5. 死亡动画处理（0.8 秒死亡动画后真正死亡）
6. 终极技能更新（包括激活状态下的持续效果）

**`applyTierBonuses(newTier)` — 位阶提升：**
- 重新计算最大生命值（基于 `CharacterProgression.hpMultiplier`）
- 更新伤害/冷却/速度倍率
- Tier 3 → 终极技能就绪
- Tier 4 → 终极技能冷却减半 + 冷却更高

**受伤机制：**
- 无敌时间内忽略伤害
- 受伤后触发 `hurtAnimTimer`（受伤动画）+ 设置无敌时间
- 死亡时设置 `dying = true` 开始死亡动画，先不设 `alive = false`

**渲染：**
- 根据动画状态选择精灵帧（IDLE/WALK/ATTACK/HURT/DEATH）
- 终极技能激活时绘制光环
- Tier 3+ 绘制位阶光环

#### **Enemy.java** (440 行) — 敌人

**10 种敌人类型：**
```
SLIME(0)    — 史莱姆，基础敌人
BAT(1)      — 蝙蝠，飞行+波浪轨迹+同类分离
SKELETON(2) — 骷髅，高血量
GIANT(3)    — 巨人，高血量，精英掉落率高
GHOST(4)    — 幽灵，远程攻击，保持距离
BOSS(5)     — Boss，半血后释放弹幕+召唤小怪
ORC(6)      — 兽人，有独立精灵动画
GOBLIN(7)   — 哥布林
MUSHROOM(8) — 蘑菇，远程攻击
FLYING_EYE(9) — 飞行眼球，远程攻击
```

**AI 行为：**

1. **基础追击：** 计算到玩家的方向向量 `(nx, ny)`，以 `baseSpeed` 追击

2. **蝙蝠特殊行为：**
   - 波浪飞行：`wobble = sin(age * 4.0) * 30°` 旋转方向
   - 同类分离：检测周围 50px 内其他蝙蝠，施加排斥力

3. **远程攻击者（GHOST/FLYING_EYE/MUSHROOM）：**
   - 保持距离：太近则后退，太远则靠近
   - 在理想距离范围内：横向移动 + 间歇射击
   - 发射投射物（`fireAtPlayer()`）

4. **Boss 阶段二（生命值 ≤ 50%）：**
   - 速度 ×1.5
   - 每 5 秒释放 8 方向弹幕（`spawnBossRadial()`）
   - 每 8 秒召唤 3 个史莱姆

**死亡掉落：**
- 掉落概率：普通 15% / 精英 30% / Boss 100%
- 掉落冷却：3 秒（`WEAPON_DROP_COOLDOWN`）
- 随机稀有度（按权重：COMMON 50 / RARE 30 / EPIC 15 / LEGENDARY 5）
- 随机武器类型（Whip / Wand / Axe / Garlic / Knife）
- 不掉落武器时掉落经验球

### 4.4 武器系统

#### **WeaponManager.java** (163 行) — 武器管理器

**核心逻辑：**

1. **`update(dt)` — 每帧更新：**

   - 冷却完毕 → `fire(w)` 发射投射物 → 重置冷却

2. **`fire(w)` — 发射武器：**

   - 已有同类型 → 升级（若未满级）
   - 有新空位 → 添加
   - 满位 → 触发武器替换界面

4. **`applyGarlicAura()` — 大蒜光环：**
   - 每 0.1 秒对范围内所有敌人造成伤害
   - 范围 = 66 + (level-1) × 12px
   - 伤害 = 基础伤害 × 0.1（持续伤害比单发低）

### 4.5 升级与成长系统

#### **UpgradeSystem.java** (166 行) — 升级系统

**经验公式：**
```
xpForNextLevel = XP_BASE(3) + level * XP_SCALAR(1.5) + level² * QUADRATIC(0.08)
```
例如：Lv1→Lv2 需要 4.58 XP，Lv50→Lv51 需要 128 XP，呈二次增长。

**升级流程：**
1. `gainXP()` → 检查是否超过阈值
2. 超过 → 扣减 XP + `level++`
3. 检测里程碑（25/50/75 级）
4. 检测位阶是否跨越 → 进入 Tier Up 或生成升级选项
5. 进入 `STATE_UPGRADE_PAUSE`，暂停游戏

**升级选项生成 `makeOneChoice()`：**
- 30% 概率：新武器（从尚未拥有的武器中随机选一个）
- 40% 概率：升级已有武器（随机选一个未满级的）
- 30% 概率：属性加成（四选一）
  - Swift Boots：移速 +10%
  - Power Core：伤害倍率 +15%
  - Magnet：拾取范围 +20%
  - Healing Light：恢复 30% HP

### 4.8 碰撞系统

#### **CollisionSystem.java** (176 行) — 碰撞检测

## 五、游戏循环详解

```
main() 
  → createGame(new SurvivalGame(), 60)
    → game.init()
      创建所有子系统、加载资源
    → game.gameLoop(60)
      → Timer.start()
        每 16.67ms 触发一次:
          dt = measureTime() / 1000  (实际帧间隔，通常 ≈ 0.0167s)
          update(dt)
            ├── handleAudioStateChange()
            ├── Boss 检测
            ├── 根据 gameState 分发:
            │   ├── STATE_PLAYING → updatePlaying(dt)
            │   │   ├── player.update(dt)     — 读取按键，计算移动
            │   │   ├── camera.follow()        — 平滑跟随
            │   │   ├── spawner.update(dt)     — 生成敌人
            │   │   ├── weaponManager.update(dt) — 自动开火
            │   │   ├── 所有 Entity.update(dt) — 位置积分
            │   │   ├── collisionSystem.update(dt) — 碰撞检测
            │   │   ├── cleanupDead()          — 移除死亡实体
            │   │   └── vfx.update(dt)         — 粒子/文字更新
            │   └── 其他状态...
            └── vfx.update(dt)  (通用)
          mPanel.repaint()
            → paintComponent()
              → 根据 gameState 渲染对应界面
```

## 七、关键设计模式

| 设计模式 | 应用位置 | 说明 |
|----------|---------|------|
| **模板方法** | `Entity.update()` final + `onUpdate()` abstract | 统一位置积分，子类自定义行为 |
| **策略模式** | `UpgradeDef.UpgradeAction` 接口 | 每个升级选项是一个策略 |
| **单例/静态工具** | `AssetLoader`, `SpriteManager`, `GameConfig` | 全局共享的无状态服务 |
| **观察者（回调）** | `Enemy.onDeath(killer)` | 死亡时触发掉落/特效/计分 |
| **状态模式** | `gameState` + switch 分发 | 11 种游戏状态的不同行为 |
| **空间哈希** | `SpatialGrid` | 碰撞检测性能优化 |
| **对象池（隐式）** | `ArrayList` + `MAX_*` 上限 | 防止无限生成导致 OOM |
| **装饰器** | `VfxManager.beginCamera/endCamera` | 相机变换 + 屏幕震动的叠加 |

## 九、潜在问题与改进建议

1. **包结构：** 所有 31 个类都在默认包中，建议分包：`engine/`, `entity/`, `system/`, `data/`, `render/`

2. **碰撞检测精度：** 圆形碰撞体对大尺寸矩形障碍物不够精确，可考虑分离轴定理（SAT）

3. **渲染顺序：** 没有深度排序（z-ordering），实体按固定顺序渲染，可能导致精灵遮挡异常

4. **音频引擎限制：** 仅支持 WAV 格式，文件较大。可考虑支持 OGG/MP3

5. **精灵动画：** Mage 类没有独立精灵动画，使用回退渲染。需补充精灵表或扩展 `SpriteManager`

6. **武器上限：** 武器数组只有 5 个槽位（`WeaponDef.ALL` 大小 20 但只用了 0-4），添加新武器需要扩展
